# Responsive and Accessibility Plan

## Mobile-First Rule

Design the portfolio so the mobile layout is the default mental model, then enhance it for larger screens.

## Responsive Priorities

- Navigation should remain usable without taking over the screen.
- Gallery items should stack cleanly and maintain readable captions.
- Section spacing should feel generous on desktop but not wasteful on mobile.
- Typography should scale down gracefully without losing the editorial feel.

## Interaction Differences

- Desktop can use hover states for artwork previews and subtle title reveals.
- Mobile should avoid hover-dependent information and show titles directly or through tap-friendly patterns.
- Primary actions must remain obvious on every breakpoint.

## Accessibility Goals

- Keep contrast strong enough for body text and labels.
- Avoid using color alone to communicate status.
- Ensure interactive elements have clear focus states.
- Use semantic sectioning so the page structure remains understandable.
- Make touch targets large enough for comfortable mobile use.

## Motion Safety

- Motion should be subtle and purposeful.
- Keep transitions short and predictable.
- Avoid large amounts of continuous animation.
- Respect reduced-motion preferences when implementation begins.
